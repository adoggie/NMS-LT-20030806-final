第一章：引言

1．1 编写目的

本手册的编制是为了向操作人员提供NMS1900直放站网管系统软件每一个运行的具体过程和有关知识，包括安装及操作细节。

1．2 背景

NMS1900直放站网管系统软件是根据《联通新时空移动通信有限公司1900MHz CDMA数字蜂窝移动通信网直放站技术规范书1.0》有关网管的规定来进行设计和开发，能管理多厂商、多型号的射频直放站和光纤直放站；同时也能非常方便地扩充对别的厂商不同类型直放站的支持。

 

产品中文名称：  NMS1900直放站网管系统

产 品 代 号：   NMS1900

版       本：   Version 1.0

1．3 定义

GSM                             全球移动通信系统(Global System Mobile)

CDMA                          码分多址(Code Division Multiple Access)

NMS                             直放站网络管理系统(Network Manage System)

LMODEM                     有线MODEM

从图（2）可以看出，网管由MSS，WSS和CSS三部分组成。

 

 

 

其中：

²        CSS

网管CSS主要负责网管信息的输入/输出，并进行原始网管信息的格式转换，给网管MSS提供统一的信息格式。网管CSS与网管ＭSS之间通过TCP连接。每个

CSS可以负责管理多个不同型号的直放站。

²        MSS

MSS是网管中的核心单元，主要负责网管信息的处理，从网管CSS接收告警等信息的输入，将其作相应的处理后，写入数据库；或者从WSS处接收对直放站的查询或控制信息，作相应处理后，经网管 CSS输出，同时写入数据库；

²        WSS

WSS是网管中心与用户交互的客户端系统，提供各种服务，用户通过客户端实现数据库的浏览访问，操作维护以及直放站的控制、设置、告警以及状态查询。

²        DB

网管 DB 是网管中心的数据库系统，主要用于存放各种数据表，包括：直放站相关信息、网管中心相关信息、直放站报警信息、直放站状态设置信息、状态值修改历史记录、用户信息（帐号、密码、权限等）、操作日志等。为了保证系统的高可靠性，可以采用双机热备份。

 

2．3 NMS1900系统功能与特点

2．3．1 系统特点

支持多种通信方式

       系统支持有线MODEM等通信方式。

支持各种类型的直放站

       系统不但支持符合联通标准协议的直放站，在提供网管数据接口的前提下通过扩充，可以管理第三方自定义协议的直放站产品。

分地区、分厂商管理功能

       根据管理员所属的区域或根据不同厂商的直放站设备进行安全有效的管理。

灵活多样的信息管理

       各种网管信息的管理采用数据库系统，可以实现数据的定时备份，根据用户要求，可定制各种方式的网管数据的统计分析，并提供多样的图表方式的显示。

 

2．3．2 系统功能

1.      状态查询功能

网管中定时对直放站进行状态轮询，获取每个直放站当前运行状态参数，管理人员可以对这些参数进行分析。当检测到有告警时，提供声,光等告警手段。轮询采用分组的方式，每组可以采用不同的轮询频率或间隔。

2.      告警管理功能

网管中心能接收直放站的主动告警，对接收到的告警提供声,光等告警手段。管理人员可以根据当前的状态参数分析告警的原因，并尽可能利用远程控制功能，排除故障。网管中心端主动告警管理用的电话号码和状态轮询用的号码一般不相同。

3.      远程控制功能

网管中心能对直放站进行远程控制，调整其运行参数。

4．权限管理

为了保证直放站网管系统的安全性，防止恶意破坏，限制无权登录和错误登录，严格控制每个操作员的操作权限。不同的权限只能使用相应的功能。并对每个操作员的操作作日志记录。

5．分区、分厂商管理

根据直放站管理中对区域的划分，将各个管理员用户和相应的区域对应起来。每个管理员只能对其所属区域内的直放站进行控制和接收主动告警；也可以为各直放站设备供应商添加管理员用户，该用户只能对其所属厂商的直放站进行控制和管理。

6．配置管理

完成网管系统中被管理的直放站数据的查询、修改、删除和添加，网管系统用户的管理，用户授权管理等。

7．日志管理

具有完整的直放站运行状态日志，告警日志，控制日志，用户登陆日志和操作日志的记录。用户可以采用模糊查询的方式进行多种方式的查询。

3．3．2 MSS配置文件说明

系统安装完成第一次运行前，必须进行系统配置，系统配置是对软件运行时所需的参数进行配置。配置文件用来对软件运行时所需的参数进行配置，该文件位于软件的安装路径下的cfg目录中，文件名为config.ini，可以用记事本打开进行编辑和修改。

下面是一个配置文件的例子：

[SNMS SERVER CONFIG]

SERVER IP=127.0.0.1

DATABASE CONNECTION=127.0.0.1

DEBUG LEVEL=10

AUTO QUERY ITEM INTERVAL=8000

为了方便说明，首先来说明一下会用到的术语：

[SNMS SERVER CONFIG]：主选项；

SERVER IP：键名；

127.0.0.1：键值；

用“=”表示给键名赋以某个键值。

键名的意义如下：

SERVER IP: Server运行机器的IP地址。

DATABASE CONNECTION: 数据库的IP地址。

DEBUG LEVEL: 调试的级别。

AUTO QUERY ITEM INTERVAL: 查询的时间间隔

3．3．3 CSS配置文件说明

配置文件用来对软件运行时所需的参数进行配置，该文件位于软件的安装路径下的config目录中，文件名为config.ini，可以用记事本打开进行编辑和修改。

下面是一个配置文件的例子：

 

[SRNMSSERVER]

ServerAddress=127.0.0.1

LocalAddress=127.0.0.1

ServerRecvPort=7000

ServerSendPort=7001

LocalRecvPort=7001

LocalSendPort=7000

CSSLabel=CSSCM

[DBSERVER]

SERVERNAME=127.0.0.1

DBNAME=NMSDB_CM

[LOGMASK]

DEBUGPRINT_FLAG=TRUE

[COMINFO]

COM1=00000,CDMASMS,RECV

COM2=00000,CDMASMS,RECV

COM3=00000,LMODEM,SEND

COM4=91004,LMODEM,SEND

COM5=00000,LMODEM,SEND

COM6=00000,LMODEM,SEND

COM7=00000,LMODEM,SEND

COM8=00000,LMODEM,SEND

COM9=91001,LMODEM,RECV

COM10=91003,LMODEM,SEND

主选项[SRNMSSERVER]中配置了有关服务器和本机的信息，具体内容包括：

ServerAddress：服务器的IP地址；

ServerRecvPort：服务器的接收端口号；

ServerSendPort：服务器的发送端口号；

LocalRecvPort：客户端的接收端口号；

LocalSendPort：客户端的发送端口号；

LocalAddress： 客户端的IP地址。

主选项[DBSERVER]中配置了CSS连接的数据库服务器名称及数据库名称参数：

SERVERNAME：数据库服务器名称；

DBNAME：数据库名称；

主选项[LOGMASK]中配置了是否输出系统日志的开关参数：

DEBUGPRINT_FLAG：TRUE为打开系统日志输出开关；FALSE为关闭系统日志输出开关。

主选项[COMINFO]中配置了关于串口信息参数：

COM1=00000,LMODEM,RECV

COM2=00000,LMODEM,SEND

…

COMx=00000,LMODEM,SEND

COMx：表示串口名称；00000：表示串口的电话号码；LMODEM：表示该串口所连接通信设备的类型（有线MODEM）；SEND：表示该串口是用于发送；RECV：表示该串口用于接收。
